2   定时器/计数器简介

2.1   基础知识

BOTTOM表示计数器最小值，一般指0x00，程序可以在溢出时修改来选择溢出周期。MAX为计数器上限，对8bit计数器为0xff，对16bit计数器为0xffff。

TCNT0与TCNT1共用一个10bit的分频器。无论选择什么时钟，都要进分频器，就是与时钟源单独选择。预分频器可以复位？

定时器的中断屏蔽寄存器TIMSK，中断标志寄存器TIFR。部分计数器，如ATmega8的T/C0和T/C1允许接受外部时钟输入，对应引脚T0和T1。T/C2不能接受外部时钟输入，但是作为RTC可以接受钟振。

输出比较寄存器OCRxn部分情况下是双缓冲的。双缓冲寄存器是设置的TOP值在当前周期内无效，在TCNTn重置后才开始奇效。这样设置TOP值比TCNTn小也没关系，反正下次过程才开始有效。双缓冲在PWM模式支持，在正常和CTC模式禁止。使用双缓冲时CPU访问缓冲寄存器，禁止双缓冲时直接访问寄存器本身。

设置输出比较时先设置COMnx，然后设置对应的DDR_COMnx。非双缓冲，立即生效。

2.3   比较匹配输出

应该先设置OC1x，然后才设置GPIO的方向寄存器DDRx。

16bit的计数器Timer2的WGM是4bit的，而不是像其他8bit计数器那样的3bit。设置时也可以设置PWM的位数。所以应用时要注意。

4   应用实例

4.1   默认时钟

对ATmega8和ATmega48，出厂默认使用内部RC振荡器，8MHz，系统默认8分频。也就是能进入定时器的时钟是1MHz。

4.2   RTC时钟

将32768Hz的钟振接到XTAL1,STAL2上，然后使用Timer2。注意将熔丝位hfuse中CKOPT由1改为0，对应ATmega8的熔丝是hfuse=0xc9，lfuse=0xe1。内部使用8MHz的RC振荡器。

计时器初始化:

TCCR2=0x00;
ASSR=1<<AS2;    //使用异步时钟
TCNT2=0xe0;     //定时1秒
TCCR2=(1<<CS22)|(1<<CS21)|(1<<CS20);    //预分频1024
TIMSK=1<<TOIE2;
sei();

Timer2的定时器溢出中断:

TCNT2=0xe0;
second++;

然后可以在main()函数中执行循环显示。

4.5   快速PWM产生功率控制

一些速度足够快，不经过MCU软件处理的PWM方式。输出极高的频率。基于ATmega328p。

以T/C0为例，PWM模式有8种，用WGM2:0(Waveform Generation Mode)来设置取值如下：

WGM2:0 操作模式 TOP 更新OCRx TOV标识置位
0 一般 0xff 立即 MAX
1 PWM相位修正 0xff TOP BOTTOM
2 CTC OCRA 立即 MAX
3 快速PWM 0xff BOTTOM MAX
4 保留 保留 保留 保留
5 PWM相位修正 OCRA TOP BOTTOM
6 保留 保留 保留 保留
7 快速PWM OCRA BOTTOM TOP

OC0A对应PD6/DIO6，OC0B对应PD5/DIO5。后面分别是AVR名字和Arduino名字。

快速PWM模式就是设置WGM=3或7，其中靠谱的是3，因为这样可以使用两个OC0x引脚做分别的输出。设置步骤：

  1. TCCR0A=COM0A1|WGM01|WGM00：TCNT0复位时，OC0x置位，TCNT0==OCR0x时OC0x清零，正相快速PWM
  2. OCR0A=比较匹配值：当TCNT0=OCR0A时，发生比较匹配，行为由COM0xx决定
  3. DDRx|=_BV(PORT)：设置OC0x引脚为输出
  4. TCCR0B=N：设置分频因子，启动定时器

4.6   CTC模式作为定期中断源

以ATmega48芯片为例。

初始化和中断:

void init_adc() {
    TCNT0=0;
    TCCR0A=_BV(WGM01);      //即WGM02:0=2，CTC模式
    TCCR0B=5;               //1024分频
    OCR0A=250;              //TOP=250
    TIMSK0=_BV(OCIE0A);
}

ISR(TIMER0_COMPA_vect) {
    if (PORTB & _BV(PB5)) {
        LED_OFF;
    } else {
        LED_ON;
    }
}

这个过程中，晶振为16MHz，经过1024分频，TOP=250后，实际为每16mS发生一次中断。用示波器实测也是这个结果。

如果要将结果输出为波形(D6/PD6/OC0A)，而不是触发中断，可以去掉 TIMSK0=_BV(OCIE0A); ，因为不必要。然后加上:

DDRD |= _BV(PD6);       //设置端口PD6为输出
TCCR0A |= _BV(COM0A0);  //比较匹配发生时翻转

5   延时AVR-GCC

使用延时之前，必须先定义两个宏，要在导入之前:

#define __OPTIMIZE__
#define F_CPU 8000000
#include <util/delay.h>

两个函数可用：

void _delay_us(double __us); ：按照微秒延时。

void _delay_ms(double __ms); ：按照毫秒延时。

这两个函数会使得代码提及增大约3300字节，非常不划算的。估计主要问题来自double类型的引入。

另外 #include <util/delay_basic.h> 提供了比较简单的延时，只能指定循环次数。引入后替代之前的循环还能使代码提及略微减小几个字节。因为是汇编实现的。

两个可用函数：

void _delay_loop_1(uint8_t __count); ：使用计数器完成延时，实际延时时间是__count*3个CPU时钟周期。

void _delay_loop_2(uint16_t __count); ：使用16bit计数器完成延时，实际延时时间为__count*4个CPU时钟周期。

引入的int32_t类型增加了程序260个字节，太可惜了。
