2   avrdude

FreeBSD下的安装:

pkg_add -r avr-binutils avr-gcc avr-libc avrdude

安装后就有了从开发的整套工具链了。而我使用的下载线是 usbasp 。

2.1   下载

将文件 demo.hex 通过USBASP下载线写入ATmega48:

sudo avrdude -p atmega48 -c usbasp -e -E noreset,novcc -U flash:w:"demo.hex"

其中-p指定目标芯片型号，-c指定数据线型号，-e指定写入前先擦除，-U后面的指定操作的文件。-E后面的两个参数表示离开下载模式时将并口的两路信号置为非reset和非vcc。这样可以不拔掉下载线而让开发板立即进入执行运行模式。

2.2   上传

将芯片中的数据写入到文件 internel.bin

sudo avrdude -p atmega48 -c usbasp -U flash:r:"internal.bin":r

估计读取芯片EEPROM数据的方法:

avrdude -p atmega48 -c usbasp -U eeprom:r:eeprom.bin:r

其他可用参数有hfuse和lfuse。

2.3   交互模式

开启进入交互模式:

sudo avrdude -p atmega48 -c usbasp -t
>>> part
AVR Part            : ATmega48
......
>>> dump lfuse
>>> dump hfuse
>>> dump efuse
>>> dump lock
>>> write hfuse 0 0x89
>>> write lfuse 0 0x2f

2.4   关于熔丝位

AVR系列有四个熔丝组，hfuse、lfuse、efuse、lock。其中前两个为满的8bit字节，efuse有些甚至没有。这些位的定义和组合可以去常用的另外一个Windows下的软件那里查询。

熔丝位定义：

  1. 已编程=Programmed=0=允许
  2. 未编程=Unprogrammed=1=禁止

不同的软件定义0和1可能有区别，要注意。

AVR芯片加密后不能读取Flash和EEPROM，芯片擦除可以清空两者的数据，同时将两者的锁定位配置为11，即无锁定。所以被锁定的芯片仍然可以恢复使用，只不过内部数据确实无法读取了，只能擦除后再写。

JTAG和ISP都可以通过熔丝来启用和禁止。RESET引脚禁用最好不要使用，因为这会使得ISP失败，进入ISP之前必须先将RESET拉低。

不用BOOTLOADER时，就不要把BOOTRST设置为0，否则启动时不是从0x0000开始的，出厂默认为1。

3   基本使用与配置

3.1   配置在Linux下不用root权限

基本思路就是使用udev。

编写文件 /etc/udev/rules.d/10-usbtinyisp.rules

SUBSYSTEM=="usb", SYSFS{idVendor}=="1781", SYSFS{idProduct}=="0c9f", GROUP="adm", MODE="0666"

然后重启udev sudo restart udev ，据说是ubuntu系统。随后重新插入usbasp编程器。

对于只有自己用的机器，还可以修改如上文件里的GROUP值为自己的组。对于多个人的机器，别人就没法用了。
