2   第一个例子

第一个例子，居然一次性成功，给我很大的鼓励。先看代码:

#include <avr/io.h>

int main(void) {
    unsigned char i,j,k,l,led=0;
    DDRB=0xff;
    while(1) {
        if(led) {
            //PORTA=0xff;
            //PORTD=0x00;
            PORTA=0xff;
            PORTB=0x00;
        }else {
            //PORTA=0x00;
            //PORTD=0xff;
            PORTA=0x00;
            PORTB=0xff;
        }
        led=!led;
        //time
        for (i=0;i<255;i++)
            for (j=0;j<255;j++)
                for (k=0;k<7;k++)
                    l++;
    }
}

再看Makefile:

MCU=atmega16

.PHONY: demo.hex

demo.hex: demo.elf
    avr-objcopy -j .text -j .data -O ihex demo.elf demo.hex

demo.elf: demo.o
    avr-gcc -mmcu=$(MCU) -o demo.elf demo.o

demo.o: demo.c
    avr-gcc -mmcu=$(MCU) -c demo.c

clean:
    rm -f *.o *.elf *.hex

wflash:
    sudo avrdude -p $(MCU) -c stk200 -e -U flash:w:"demo.hex"

rflash:
    sudo avrdude -p $(MCU) -c stk200 -U flash:r:"rflash.hex"

4   Linux下的AVR开发环境

发行版Debian Sarge，桌面Gnome 2.10.2。

Gcc环境：binutils-avr_2.15-3、gcc-avr_3.4.3-2、avr-libc_1.2.3-3。

avrdude是源码编译的，不过现在Debian源里面也有了。

EDA环境使用geda_20050313-2版，主要使用gschem画草图，也可以生成PCB layout。

编程器用的AVRISP线，可以在线编程，而且可选发送reset，直接运行起来。

5   一个较为通用的AVR的Makefile模板

已经下载到现在的目录下的 Makefile.tpl 了。其中按照你的项目需要修改的只有三处:

MCU=at90s8515       #修改目标控制器型号
PROJECTNAME=myproject   #项目名字
PRJSRC=main.c myclass.cpp lowlevel.S    #项目源代码

其他附加参数设置各种路径，编程器(avrdude)等等。默认编程线是STK500，需要修改。

make ：构建你的项目，编译、汇编和连接到ELF格式的 myproject.out 。

make hex ：生成用于上传到芯片的程序，会生成 myproject.hex 和 myproject.ee.hex ，基于ELF .out文件。两个文件都是Intel HEX格式的，分别用于写入到flash和eeprom。

有这两个文件以后可以用 make writeflash 来写入到芯片，或者手动执行，比如STK500的编程器:

$ avrdude -p 8515 -c stk500 -e -U flash:w:myproject.hex

make disasm 查看编译过程，了解编译出的代码，提供代码段信息。会生成 myproject.s 文件。

make *.s 用于检查汇编输出。自动生成的汇编文件用小写的".s"，会在 make clean 时清掉，所以你自己的汇编文件记得用".S"。

make writeflash 使用avrdude将程序写入芯片。

make gdbinit 带调试标志的编译，用于之后的模拟和调试。生成 gdbinit-target 。调试:

$ avr-gdb -x gdbinit-myproject

make clean 清除编译中间结果。
