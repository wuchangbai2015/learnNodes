1   简介

SPI是个很不错的接口方式，尤其是很省资源，现在连ATtiny都内置了。

与SPI相关的芯片有如下几种：

  1. 74HC595：用于扩展输出端口，串入并出
  2. 74HC165：用于扩展输入端口
  3. DS1302：一种时钟芯片

2   AVR的SPI接口

2.1   SPI简介

特点：

  1. 全双工，3线同步数据传输
  2. 主机或从机操作
  3. LSB首先发送或MSB首先发送
  4. 7种可编程的比特率
  5. 传输结束中断标志
  6. 写冲突标志检测
  7. 可以从闲置模式中环形
  8. 作为主机时具有倍速模式(CK/2)

主机和从机的通信，主机发送出去的是MOSI，从机发送出去的是MISO，同时双方还要接SCK作为移位时钟。/SS用于启动和结束通信，/SS拉低时开始通信，拉高时结束通信。注意/SS是由用户控制的，而不是SPI自动。

主机对SPI数据寄存器写入数据就启动SPI时钟，传输结束后SPI时钟停止，并置位SPIF。如果此时SPCR寄存器SPIE位使能，则产生中断。主机可以继续向SPDR写入数据，或者将从机/SS拉高说明数据传送完成，最后进来的数据将一直保存于缓冲寄存器中。

从机可以随意写入SPDR，不过只要/SS为高电平就不会通信，即使有SCK时钟。直到/SS拉低，然后随着SCK时钟开始通信。1个字节移出后，传输结束标志SPIF置位，如果此时SPCR的SPIE置位，则产生中断。通信时实际发送的是最新的SPDR中的数据，多次写入也无所谓，之前的都被覆盖了。

通信过程中不能对SPDR读写，因为此时数据没意义。发送方只需要一个缓冲器，而接收方则需要两个，一个用于接收。得知数据收到后应该立即提取数据，否则过一会可能会被覆盖。

SPI相关的4个引脚：MOSI、MISO、SCK、/SS，的输入输出情况：

引脚 SPI主机 SPI从机
MOSI 用户定义 input
MISO input 用户定义
SCK 用户定义 input
/SS 用户定义 input

从机使用/SS作为输入，低电平时通信有效。用于数据包的字节同步。在拉低时可以连续发送多个字节。

主机使用/SS作为输出，控制从机的通信同步。如果/SS配置为输入且被外部拉低，则SPI系统会自动将自己变成从机，防止冲突。步骤如下：

  1. 清零SPCR的MSTR位，使SPI变成从机，而MOSI和SCK变成输入
  2. SPSR的SPIF置位，如果SPI中断和全局中断使能，则会产生中断

一般情况下不要让主机的/SS作为输入，而是作为输出，由用户控制。不过也可以自己控制作为多机通信系统。中断方式处理SPI数据时，应该检查MSTR是否为1，如果被清零，则用户必须重新置位，使能SPI主机模式。

SPCR中CPOL和CPHA位控制起始沿和结束沿与模式：

CPOL,CPHA 起始沿 结束沿 SPI模式
00 采样,上升沿 设置,下降沿 0
01 设置,上升沿 采样,下降沿 1
10 采样,下降沿 设置,上升沿 2
11 设置,下降沿 采样,上升沿 3

对ATmega8/ATmega48/88/168来说的引脚映射：

  1. SCK => PB5
  2. MISO => PB4
  3. MOSI => PB3
  4. /SS => PB2

2.2   通信例程

注意DDR_SPI要替换成实际的DDR寄存器。

初始化SPI主机和发送:

#define DDR_SPI DDRB
#define DD_SS   PB2
#define DD_MOSI PB3
#define DD_MISO PB4
#define DD_SCK  PB5
#define SET_SS  PORTB |= _BV(PB2);  //这里对SS的操作供74HC595使用
#define CLR_SS  PORTB &= ~_BV(PB2);

void SPI_MasterInit(void) {
    //设置MOSI和SCK为输出，其他使用默认输入
    DDR_SPI=(1<<DD_MOSI)|(1<<DD_SCK)|(1<<DD_SS);
    //使能SPI主机模式，时钟速率为fck/16
    SPCR=(1<<SPE)|(1<<MSTR)|(1<<SPR0);
}

uint8_t SPI_SendRecv(uint8_t c) {
    CLR_SS;
    SPDR=c;
    while(!(SPSR & (1<<SPIF)))
        ;
    c=SPDR;
    SET_SS;
    return c;
}

初始化SPI从机和接收:

void SPI_SlaveInit(void) {
    //设置MISO为输出，其他使用默认输入
    DDR_SPI=(1<<DD_MISO);
    SPCR=(1<<SPE);
}

char SPI_SlaveReceive(void) {
    //等待接收结束
    while(!(SPSR & (1<<SPIF)))
        ;
    return SPDR;
}

对ATmega8/ATmega48/88/168来说的预定义:

#define DDR_SPI     DDRB
#define DD_SCK      PB5
#define DD_MISO     PB4
#define DD_MOSI     PB3
#define DD_SS_      PB2

因为SPI是同时通信的，所以更推荐的通信函数是双向的，尽管可以先抛弃接收部分:

char SPI_SendRecv(char cData) {
    SPDR=cData;
    while(!(SPSR & (1<<SPIF)))
        ;
    return SPDR;
}

2.3   相关寄存器

SPI控制寄存器-SPCR ： [7:SPIE][6:SPE][5:DORD][4:MSTR][3:CPOL][2:CPHA][1:SPR1][0:SPR0] ，各个位含义：

  1. [7:SPIE]{RW/0} ：SPI中断使能
  2. [6:SPE]{RW/0} ：SPI使能
  3. [5:DORD]{RW/0} ：数据次序，置位时LSB先发送，清零时MSB先发送
  4. [4:MSTR]{RW/0} ：主机/从机模式，置位为主机；若主机时/SS为输入但被拉低，则MSTR会被清零，SPSR的SPIF置位，用户需要重新置位MSTR进入主机模式
  5. [3:CPOL]{RW/0} ：时钟极性，置位表示空闲时SCK高电平，起始沿是下降沿；清零表示空闲时SCK低电平，起始沿为上升沿
  6. [2:CPHA]{RW/0} ：时钟相位，决定起始沿还是结束沿对SCK采样，置位则在起始沿设置，结束沿采样；清零则起始沿采样，结束沿设置
  7. [0,1:SPR1,SPR0]{RW/0} ：用于设置SPI时钟，由主机设置，从机无视

SPI时钟选择：

SPR1:SPR0 SCK频率
00 Fosc/4
01 Fosc/16
10 Fosc/64
11 Fosc/128

如果SPI2X置位，则上面的频率加倍，比如SPR1:0=00时变成Fosc/2。

SPI状态寄存器-SPSR ： [7:SPIF][6:WCOL][5:-][4:-][3:-][2:-][1:-][0:SPI2X] ，各个位定义：

  1. [7:SPIF]{R/0} ：SPI中断标志。串行发送结束就置位，如果SPI主机/SS输入被拉低也会置位。进入中断程序会自动清零，也可以先读SPSR，然后访问SPDR来清零
  2. [6:WCOL]{R/0} ：写冲突标志，发送中对SPDR写数据则置位，可以先读SPSR，再访问SPDR来清零
  3. [0:SPI2X]{RW/0} ：SPI倍速模式。如果为主机最高可以到CPU频率一半，从机最高是Fosc/4

ATmega8的SPI接口同时实现Flash和EEPROM的上传下载，详见SPI串行编程和校验。

SPI数据寄存器-SPDR ： [7:MSB][6:-][5:-][4:-][3:-][2:-][1:-][0:LSB] ，就是一个读写寄存器。写入将启动数据传输，读取则是读接收缓冲器。

3   74HC595

8bit串入并出移位寄存器，电压范围2~6V，80uA工作电流，典型传输时延14nS，5V时可以驱动6mA电流。移位寄存器可以直接清除。

5个信号输入端：

  1. /OE,OE# ：低电平时，输出Qa到Qh锁存器数据，高电平时Qa到Qh高阻抗
  2. ST_CP,RCLK ：锁存器时钟，上升沿移位寄存器中所春数据加载到输出锁存器
  3. /MR,SRCLR# ：移位寄存器清零，低电平有效
  4. SH_CP,SRCLK ：移位寄存器时钟，上升沿时，移位寄存器数据向左(高位)移动1位
  5. DS,SER ：移位寄存器串行数据输入端，在SRCLK指导下依次进入移位寄存器
  6. Q7' ：串行输出

DIP引脚排布:

Q1  1    16  Vcc
Q2  2    15  Q0
Q3  3    14  DS
Q4  4    13  /OE
Q5  5    12  ST_CP
Q6  6    11  SH_CP
Q7  7    10  /MR
GND 8    9   Q7'

连接AVR系列的SPI接口时，按照如下连接：

  1. SS => ST_CP,RCLK：作为输出使能控制
  2. SCK => SH_CP,SRCLK：移位寄存器时钟
  3. MOSI => DS,SER：移位寄存器串行输入
  4. /OE => GND
  5. /MR => Vcc

当多个74hc595串接时，SS和SCK都是一样并联的，然后上一个芯片的Q7'接下一个芯片的SER。

实际发送数据时，需要先将/SS拉低，发送完再把/SS拉高。

4   74HC165

8bit并入串出移位寄存器。输出有Q7和/Q7两种。

当/PL为低电平时，D0到D7的并行数据被异步载入到寄存器。当/PL为高电平时，寄存器中的数据串行的按照从Q0->Q1->Q2的顺序输出，按照正向时钟。该功能使得多个并入串出转换器可以通过Q7到DS的方式串接扩展。

/PL低电平时，数据从D0到D7载入到寄存器。/PL为高电平时，数据已经锁存，供输出。此时若/CE为低，CP有时钟，则数据从Q7和/Q7输出。

时钟输入CP与片选是通过门控或输入的。CP和/CE引脚可以随意选用，供布局的方便。/CE的上升沿相当于CP的高电平预期操作。因为是或门，所以/CE为高电平(片选无效)时，无论CP输入什么，都相当于输入时钟为一直的高电平，无效。而当/CE为电平时，CP才能有效进入芯片提供时钟。

进入CP或/CE的高电平前，应该让/PL经历上升沿，以便让数据进入移位寄存器。

DIP引脚排布：

/PL 1 16 Vcc
CP 2 15 /CE
D4 3 14 D3
D5 4 13 D2
D6 5 12 D1
D7 6 11 D0
/Q7 7 10 DS
GND 8 9 Q7

从RPi实验所得的连接方法：

  1. 先确保外部主机片选信号为空闲高电平，有效低电平
  2. 外部片选接入/CE(15)
  3. 外部时钟接入CP(2)
  4. 外部的MISO接入Q7(9)
  5. DS供串接扩展，可接地，或接入下一级的Q7
  6. 外部片选通过反相器转换后接/PL(1)
  7. /Q7(7)悬空，Vcc(16)接电源，Gnd(8)接地线
  8. 反相器可用NPN三极管做，发射极接地，基极通过1Kohm电阻输入，集电极通过10Kohm电阻接Vcc，集电极作输出
