2   Maple简介

面向物理计算平台的。他们的公司提供各种电子设计服务。

2.1   已有的设备及性能

2.1.1   Maple

发布于2009年，兼容Arduino引脚，使用GCC编译器。LeafLabs编写了C库(libmaple)给这个板子。2011年出过少量的RET6版本。2014年仍然有制造，不过不打算继续升级了。

  1. STM32F103RB：Cortex-M3内核，72MHz
  2. 20KB RAM，128KB Flash
  3. 专用的USB口供编程和通信
  4. 43个GPIO
  5. 15个模拟输入，12bit分辨率
  6. 15个PWM输出，16bit分辨率
  7. 2个I2C和SPI口
  8. 3个USART串口
  9. 7通道DMA
  10. 低功耗和睡眠模式
  11. 集成LiPo电池充电
  12. 尺寸2.05" x 2.1"

详细文档： http://leaflabs.com/docs/hardware/maple.html

2.1.2   Maple Mini

发布于2011年早期，是一个可兼容面包板的PCB。在2014年仍然对发布者有效，会继续集成，不过不会发布新的版本，和移植新的OS。

  1. STM32F103RCBT6：Cortex-M3，72MHz
  2. 20KB RAM，128KB Flash
  3. 专用的USB口供编程和通信
  4. 34个GPIO
  5. 9个模拟输入，12bit分辨率
  6. 12个PWM输出，16bit分辨率
  7. 2个I2C和SPI口
  8. 3个USART串口
  9. 7通道DMA
  10. 低功耗和睡眠模式
  11. 尺寸2.02" x 0.72"

详细文档： http://leaflabs.com/docs/hardware/maple-mini.html

2.1.3   Maple Native

提供功能更加强大的产品。2011年秋季发布，支持libmaple。

  1. STM32F103ZET6，Cortex-M3，72MHz
  2. 64KB+1MB RAM，512KB Flash
  3. 专用的USB口供编程和通信
  4. 106个GPIO
  5. 21个模拟输入，12bit分辨率
  6. 2个DAC，12bit分辨率
  7. 17个PWM输出，16bit分辨率
  8. 2个I2C，3个SPI
  9. 5个U(S)ART串口
  10. 12通道DMA
  11. 高密度FSMC引出
  12. 低功耗和睡眠模式
  13. 集成LiPo电池充电
  14. 尺寸4" x 2.1"

详细文档： http://leaflabs.com/docs/hardware/maple-native-beta.html

2.1.4   其他短暂的设备

曾有Maple II，基于STM32F407ZE的。

Oak产品线计划合并低端FPGA，250K门的Xilinx Spartan 3E，作为全功能处理平台适合机器人，机器视觉、监控、游戏、其他主机应用。对于不熟悉FPGA的用户，LeafLabs会提供预编译的FPGA核心(包括C接口)来实现常见任务。已有硬件设计，但还没有软件支持。

Willow集成了高性能ARM核心和FPGA板，适合实时机器视觉，算法加速，机器人控制等。现在正在研究nVidia Tegra处理器，不过可能要几年。

3   底层组件以及玩法

3.1   GCC编译器

需要使用 arm-none-eabi-gcc 。CodeSourcery版本的GCC。

Maple IDE里已经附带了arm-none-eabi-gcc，一般不需要直接调用。

默认的编译参数:

-Os -g -mcpu=cortex-m3 -mthumb -march=armv7-m -nostdlib
-ffunction-sections -fdata-sections -Wl,--gc-sections
另外的C++参数::
-fno-rtti -fno-exceptions -Wall

传递给汇编器的:

-mcpu=cortex-m3 -march-armv7-m -mthumb

C库默认是newlib，而不是libc。

3.2   Unix工具链

需要预先安装的工具:

sudo apt-get install build-essential git-core wget screen dfu-util openocd python python-serial

dfu-util来自OpenMoko项目，用于通过USB上传程序。需要0.6或更高版本。Debian 8默认的是0.5版本。

获取libmaple和编译:

$ cd ~
$ git clone git://github.com/leaflabs/libmaple.git libmaple
$ cd libmaple
$ wget http://static.leaflabs.com/pub/codesourcery/gcc-arm-none-eabi-latest-linux32.tar.gz
$ tar xzvf gcc-arm-none-eabi-latest-linux32.tar.gz
$ export PATH=$PATH:~/libmaple/arm/bin

安装udev规则，先运行 groups ，然后确保输出包含plugdev和dialout，如果自己添加进去:

$ sudo usermod -a -G plugdev $USER
$ sudo usermod -a -G dialout $USER

然后注销并重新登录来生效。随后安装udev规则:

$ sudo cp support/script/45-maple.rules /etc/udev/rules.d/45-maple.rules
$ sudo restart udev

作为安全措施，未知USB设备只能由root访问。这个udev规则是的Maple可以挂载到 /dev/maple ，并且授予 plugdev 组以读写权限。在重启udev后，需要重新插拔来生效。

3.3   自行编译和上传

进入libmaple目录，来测试工具链:

$ cd ~/libmaple
$ cp main.cpp.example main.cpp
$ make clean
$ make

如果你给另一个板子做编译，你需要设置 BOARD 环境变量。例如Maple Mini的两种方法:

$ export BOARD=maple_mini
$ make
$ BOARD=maple_mini make

执行正常则会看到sizei信息。

上传通过USB串口，Linux下可以直接执行 make install 。

3.4   调试与通信

USB串口可以是 /dev/ttyACM<xxx> 或 /dev/tty.usbmodem<xxx> 。

启用 screen /dev/tty<xxx> 。Mac退出 Ctrl-a Ctrl-\ 。Linux退出 Ctrl-a k。

可以将libmaple编译为静态链接文件 make library 。会创建 ~/libmaple/build/libmaple.a 。
