2   板子简介

官方的Micro是有ISP接口的，可惜我买的淘宝的是没有这个接口的，估计仅仅为了做的更小更便宜。

一共20个GPIO，其中7个PWM，12个ADC。16MHz 5V。

类似于Arduino Leonardo，因为芯片一样。内置USB通信，不再需要附加芯片。这使得Micro可以作为电脑的鼠标、键盘、CDC串口等。

问题是我手里的Micro与官方的Micro几乎完全不同，少了很多引脚。我都怀疑是否还能通过SPI编程了。

3   输出针脚

3.1   淘宝的Micro

那种便宜的27元货。以USB口朝上：

TXO/PD3 RAW稳压输入
RXI/PD2 GND
GND RST
GND Vcc
D2/SDA/PD1 A3/ADC4/PF4
D3/SCL/PD0 A2/ADC5/PF5
D4/ICP1/ADC8/PD4 A1/ADC6/PF6
D5/OC3A/#OC4A/PC6 A0/ADC7/TDI/PF7
D6/T0/OC4D/ADC10/PD7 D15/SCK
D7/INT6/AIN0/PE6 D14/MISO
D8/ADC11/PB4 D16/MOSI
D9/ADC12/PB5 D10/ADC13/PB6

有一堆的东西与官方的不同，既有少的也有多的。

RAW是稳压输入。J1如果被短路就不经过稳压的MIC5219了，用于5V，断开用于3.3V。

PD5低电平点亮绿色LED。PB0/SS低电平点亮黄色LED。

好吧，至少可以通过SPI来编程的，外部接口支持非常贫瘠。差一点让我动手去研究Leonardo了。

官方版本的D11、D12、D13、A4、A5都没有包含。

制造商提供的测试程序:

int RXLED=17;
//TXLED是预定一的宏TXLED1、TXLED0

void setup() {
    pinMode(RXLED, OUTPUT);
    Serial.begin(9600); //连接串口监控器
    Serial1.begin(9600);//UART，连接外部传感器
}

void loop() {
    Serial.println("Hello world");  //串口0只能与电脑通信
    Serial1.println("Hello!");      //通过硬件UART发送
    //点亮2个灯
    digitalWrite(RXLED, HIGH);
    TXLED1; //TX LED未绑定到一般控制引脚
    delay(1000);
    //熄灭2个灯
    digitalWrite(RXLED, LOW);
    TXLED0;
    delay(1000);
}
